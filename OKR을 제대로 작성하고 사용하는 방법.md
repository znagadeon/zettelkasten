# OKR을 제대로 작성하고 사용하는 방법

## OKR이란

> Objective: 이전에 없었던 새로운 시장을 개척한다.
>
> Key Result 1: 30~40대 신규 고객 2만명
> Key Result 2: 신규 서비스 A 매출 20억
> Key Result 3: 신규 서비스 A 고객만족도 4.8

- Objective + Key Results
- Objective: 목표에 대한 **정성적인** 설명
  - 조직원들에게 영감을 주고 동기부여가 되도록
  - 정량적인 수치를 사용하지 않도록 주의하자
    - 수치로 동기부여가 될 수도 있겠지만, 대개는 그렇지 않음
- Key Results: Objective를 달성했다는 것을 알려주는 **구체적인 수치**
  - 만약 Key Results를 만족했다면, Objective를 달성한 것
  - 수치로 표현하기 어려운 결과이더라도, 되도록 수치로 표기하는 것이 좋다
    - ex) 고객 만족도 90%
    - 수치로 표현해야만 측정할 수 있고 관리가 가능해진다

## 작성방법

### Objective

> 1. 우리 회사의 미션을 달성하기 위해 이 기간에 도전하고 싶은 것은?
> 2. 구성원들은 무엇을 위해 일하고 싶어하는가?

### Key Results

> 1. 목표를 달성했다는 것을 알려주는 구체적인 수치는 무엇인가?
> 2. Key Results를 모두 달성했다면 Objective가 달성되었다고 볼 수 있는가?
> 3. 현재 상태에서 열심히 했을 때 70% 정도 달성 가능한 도전적인 결과는 무엇인가?

- 어떤 면에서는 KPI와 비슷하다
- Key Results와 Action Item을 혼동하지 않도록 주의하자
  - 신규 시스템 런칭하기 (X)
  - 신규 시스템 오류 5% 미만 (O)

## 왜 어려운가?

- 정성적인 Objective를 만들어내기 어렵다
  - 동기부여가 되거나 지금 하는 일의 의미를 생각해본 적이 거의 없다
  - 어려운 일이지만, 이것을 고민하는 것 자체가 일에 대한 몰입도나 주도성을 올려준다
- Key Results로 사용할 만한 지표를 찾기가 어렵다
  - 최대한 자신의 업무를 **측정 가능한** 상태로 만들어보자
- 잘 쓴 건지 알기가 힘들다

## 예시

> Objective: 깜짝 놀랄 만한 웹사이트를 런칭한다
>
> Key Result 1: 새로운 디자인 시안 3개를 만든다
> Key Result 2: 웹사이트 개발자 2명을 채용한다
> Key Result 3: 새로운 웹사이트를 2월까지 오픈한다

### 문제점

- Key Results가 아니라 Action Item(Initiative)을 작성하였음
  - Initiative를 통해 만족하고 싶은 결과를 작성하자
- Key Results가 모두 달성되었다고 해서 Objective가 달성되었다고 할 수 없음
- "깜짝 놀랄 만한"이 가진 의미가 모호함

### 고쳐 보기

- 기존의 Key Results를 Initiatives로 수정하고, Initiatives로 달성할 수 있는 Key Results를 새로 만든다
  - ex) 방문 지속시간 100% 상승, 구매 전환율 30% 상승
- Key Results를 기반으로 Objective의 모호성을 제거한다
  - 더 많은 구매 전환을 일으키는 높은 품질의 웹사이트

> Objective: 더 많은 구매 전환을 일으키는 높은 품질의 웹사이트를 런칭한다
>
> Key Result 1: 방문 지속시간 100% 상승
> Key Result 2: 구매 전환율 30% 상승
>
> Initiative 1: 새로운 디자인 시안 3개를 만든다
> Initiative 2: 웹사이트 개발자 2명을 채용한다
> Initiative 3: 새로운 웹사이트를 2월까지 오픈한다
